# Codex.app 26.623.81905 Build 4598 Adaptation Design

## Context

The installed `/Applications/Codex.app` reports `CFBundleShortVersionString` `26.623.81905` and `CFBundleVersion` `4598`.

Direct extraction of `Contents/Resources/app.asar` and `pnpm inspect:bundle-targets` showed that the existing 26.623 Fast, service-tier, `/fast`, Intelligence Speed, and settings schema target families still locate their guarded targets. The `Disable automatic updates` General settings row moved to a new minified hook shape in `general-settings-BMV5mko2.js`, and the GPT-5.5 model-list bridge moved to a new `$7((e,t)=>e.sendRequest(...))` wrapper shape, so both targets needed narrow matcher updates.

## Approach

Add support for `26.623.81905+4598` as another exact 26.623-family build, but only after adding a narrow Settings-row target for the new `La()` shape. Reuse the existing `GENERAL_SETTINGS_REPLACEMENT_26623` callback so the injected row keeps `codexfast*` local names and avoids the prior Settings TDZ failure mode.

Keep Plugins runtime targets skipped for this exact build because the inspected 26.623 family continues to use the official Plugins path for this repo's target use case.

## Scope

- Add regression fixtures for the new General settings row shape and GPT-5.5 model-list wrapper shape.
- Add the new row matcher in `src/targets/updates.mts` and broaden the direct model-list wrapper local in `src/targets/models.mts`.
- Add `26.623.81905+4598` to supported-version and runtime Plugins-skip gates.
- Update compatibility docs, patch-target docs, troubleshooting/validation lists, and a bundle note.
- Regenerate `bin/codexfast`.

## Validation

- Red test: the new General settings row fixture and `$7` model-list fixture must initially fail to patch.
- Green test: the same fixtures must patch to `disableAutomaticUpdates`, avoid copied minified locals, and preserve GPT-5.5 Fast service-tier metadata injection.
- `pnpm build:check`
- `pnpm typecheck`
- `pnpm check:version-drift`
- `bash test/re-sign-flow.sh`
- `pnpm test`
- If feasible, run a real installed-app `launch` pass and verify app bundle files remain unchanged.
